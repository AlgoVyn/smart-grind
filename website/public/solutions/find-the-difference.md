# Find The Difference

## Problem Description

You are given two strings s and t.
String t is generated by random shuffling string s and then add one more letter at a random position.
Return the letter that was added to t.

## Examples

**Example 1:**

**Input:** s = "abcd", t = "abcde"

**Output:** "e"

**Explanation:** 'e' is the letter that was added.

**Example 2:**

**Input:** s = "", t = "y"

**Output:** "y"

## Constraints

- 0 <= s.length <= 1000
- t.length == s.length + 1
- s and t consist of lowercase English letters.

---

## Pattern:

This problem follows the **Bit Manipulation - XOR** pattern for finding a single difference between two similar sets.

### Core Concept

- **XOR Cancellation**: `a ^ a = 0` cancels pairs, `a ^ 0 = a` preserves single element
- **Character Encoding**: Convert characters to ASCII/Unicode values for XOR operation
- **Single Pass**: Process both strings together to find the extra character

### When to Use This Pattern

This pattern is applicable when:
1. Finding elements that appear differently between two arrays/strings
2. Problems requiring O(1) space solution
3. Detecting single differences in paired/cancelled data

### Related Patterns

| Pattern | Description |
|---------|-------------|
| XOR Finding | Find single element among pairs |
| Sum/Difference | Alternative using arithmetic |
| Hash Map | Frequency counting approach |

---

## Intuition

The key insight is that we need to find the single character that was added to string s to create string t. Since all characters in s appear in t (just shuffled) plus one extra character, we can use the XOR operation to find the difference efficiently.

The XOR operation has two crucial properties:
- `a ^ a = 0` (any number XORed with itself is 0)
- `a ^ 0 = a` (any number XORed with 0 is itself)

By XORing all characters in both strings, pairs of identical characters will cancel out, leaving only the extra character.

---

## Multiple Approaches with Code

We'll cover three approaches:

1. **XOR Approach** - Optimal O(n) time, O(1) space
2. **Sum Approach** - O(n) time, O(1) space
3. **Hash Map Approach** - O(n) time, O(n) space

---

## Approach 1: XOR Approach (Optimal)

This is the most efficient approach using the XOR bitwise operation.

### Why It Works

When we XOR all characters from both strings, every character that appears the same number of times in s and t will cancel out (become 0). The character that appears one more time in t will remain as the result.

### Code Implementation

````carousel
```python
class Solution:
    def findTheDifference(self, s: str, t: str) -> str:
        """
        Find the difference between two strings using XOR.
        
        Args:
            s: Original string
            t: String with one extra character
            
        Returns:
            The extra character added to t
        """
        result = 0
        # XOR all characters from both strings
        for c in s + t:
            result ^= ord(c)
        return chr(result)
```
````

## Approach 2: Sum Approach

This approach calculates the difference between the ASCII sums of both strings.

### Why It Works

The sum of all characters in t minus the sum of all characters in s equals the extra character.

### Code Implementation

````carousel
```python
class Solution:
    def findTheDifference_sum(self, s: str, t: str) -> str:
        """
        Find the difference between two strings using sum.
        
        Args:
            s: Original string
            t: String with one extra character
            
        Returns:
            The extra character added to t
        """
        sum_s = sum(ord(c) for c in s)
        sum_t = sum(ord(c) for c in t)
        return chr(sum_t - sum_s)
```
````

## Approach 3: Hash Map Approach

This approach counts character frequencies and finds the extra character.

### Why It Works

By counting occurrences of each character in s and t, we can identify which character has a count of 1 in t but 0 in s.

### Code Implementation

````carousel
```python
from collections import Counter

class Solution:
    def findTheDifference_hashmap(self, s: str, t: str) -> str:
        """
        Find the difference using hash map.
        
        Args:
            s: Original string
            t: String with one extra character
            
        Returns:
            The extra character added to t
        """
        count = Counter(t)
        for c in s:
            count[c] -= 1
        for c, freq in count.items():
            if freq == 1:
                return c
        return ''
```

<!-- slide -->
```cpp
#include <unordered_map>
class Solution {
public:
    char findTheDifference(string s, string t) {
        unordered_map<char, int> count;
        for (char c : t) count[c]++;
        for (char c : s) count[c]--;
        for (auto& pair : count) {
            if (pair.second == 1) return pair.first;
        }
        return ' ';
    }
};
```

<!-- slide -->
```java
import java.util.*;
class Solution {
    public char findTheDifference(String s, String t) {
        Map<Character, Integer> count = new HashMap<>();
        for (char c : t.toCharArray()) {
            count.put(c, count.getOrDefault(c, 0) + 1);
        }
        for (char c : s.toCharArray()) {
            count.put(c, count.get(c) - 1);
        }
        for (Map.Entry<Character, Integer> entry : count.entrySet()) {
            if (entry.getValue() == 1) return entry.getKey();
        }
        return ' ';
    }
}
```

        count[c]--;
    }
    for (const c in count) {
        if (count[c] === 1) return c;
    }
    return '';
};
````

## Follow-up Questions

### Q1: Can you solve it without using any extra space?

**Answer:** Yes! The XOR approach uses O(1) extra space - only one variable is needed regardless of input size.

---

### Q2: What if the strings contain uppercase letters or special characters?

**Answer:** The XOR and sum approaches work for any character encoding. The hash map might need to handle a larger character set, but still uses O(1) space since character space is bounded.

---

### Q3: How would you modify the solution to handle Unicode characters?

**Answer:** The XOR approach works with Unicode since it operates on character codes. For the sum approach, use a larger integer type to avoid overflow. The hash map approach naturally handles Unicode.

---

### Q4: What edge cases should be tested?

**Answer:**
- Empty string s with single character t
- Strings with all same characters
- Strings with one character difference
- Maximum length strings

---

### Q5: Why is XOR preferred over subtraction for finding the difference?

**Answer:** XOR is preferred because:
- No risk of integer overflow (subtraction could overflow for large values)
- More elegant and mathematically interesting
- Demonstrates bitwise operation knowledge
- Works correctly with negative numbers if they appear

## Common Pitfalls

### 1. Not Converting to Ordinal Values
**Issue:** XORing characters directly without converting to ordinal values.

**Solution:** Use `ord()` in Python, `charCodeAt()` in JavaScript, or cast to int in Java/C++.

### 2. String Concatenation Order
**Issue:** Concatenating strings incorrectly for XOR operation.

**Solution:** `s + t` works but iterate separately for clarity.

### 3. Sum Overflow
**Issue:** Sum approach can overflow for very long strings.

**Solution:** Use XOR approach which has no overflow risk.

### 4. Character Type Mismatch
**Issue:** Returning wrong type (int instead of char).

**Solution:** Convert back using `chr()` in Python, `String.fromCharCode()` in JS.

The **Find The Difference** problem demonstrates the power of XOR operations for finding a single difference between two similar sets:
