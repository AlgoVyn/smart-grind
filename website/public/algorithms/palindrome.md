# Palindrome (Number)

## Category
Mathematics & Number Theory

## Description

A palindrome number reads the same backward as forward. Checking for numeric palindromes involves comparing digits from both ends without converting to string (though string conversion is also valid). The mathematical approach provides O(1) space complexity by reversing only half of the digits, making it efficient for large numbers.

Palindrome numbers have interesting mathematical properties and appear frequently in coding interviews and competitive programming. They can be generated by mirroring halves, counted using digit dynamic programming, and used in various number-theoretic problems. Understanding both the numeric and string-based approaches is valuable for different constraint scenarios.

## Concepts

The palindrome pattern relies on fundamental number manipulation and symmetry concepts.

### 1. Palindrome Detection Methods

Different approaches to check if a number is palindrome:

| Method | Time | Space | Best For |
|--------|------|-------|----------|
| **Reverse Half** | O(log n) | O(1) | Large numbers, space constrained |
| **Full Reverse** | O(log n) | O(1) | Simple implementation |
| **String Conversion** | O(log n) | O(log n) | Quick, readable code |
| **Recursive** | O(log n) | O(log n) | Educational purposes |

### 2. Digit Properties

Key observations about numeric palindromes:

| Property | Description | Example |
|----------|-------------|---------|
| **Negative** | Never palindromes | -121 → false |
| **Trailing Zero** | Can't be palindrome (except 0) | 10 → false |
| **Digit Count** | Even or odd both possible | 1221 (even), 121 (odd) |
| **Divisibility** | Even-digit palindromes divisible by 11 | 1221 / 11 = 111 |

### 3. Palindrome Generation

Creating palindromes from half:

| Type | Generation | Example |
|------|------------|---------|
| **Even Length** | Mirror entire left half | "12" → "1221" |
| **Odd Length** | Mirror left half excluding middle | "12" → "121" |
| **Next Palindrome** | Increment half, handle carry | 129 → 131 |

### 4. Counting Palindromes

Number of palindromes up to N:

| Digits | Count Formula | Example (≤ 1000) |
|--------|---------------|------------------|
| **1 digit** | 9 | 1-9 |
| **2 digits** | 9 | 11, 22, ..., 99 |
| **3 digits** | 90 | 101, 111, ..., 999 |
| **Total** | Roughly 2 × √N | ~19 for N=100 |

## Frameworks

Structured approaches for palindrome-related problems.

### Framework 1: Reverse Half Comparison

```
┌─────────────────────────────────────────────────────────────┐
│  REVERSE HALF PALINDROME CHECK                                │
├─────────────────────────────────────────────────────────────┤
│  Input: Integer x                                            │
│  Output: True if palindrome, False otherwise                  │
│                                                               │
│  1. Handle edge cases:                                         │
│     - If x < 0: return False                                 │
│     - If x % 10 == 0 and x != 0: return False                │
│                                                               │
│  2. Initialize reversed_half = 0                              │
│                                                               │
│  3. While x > reversed_half:                                  │
│     - Extract last digit: digit = x % 10                       │
│     - Append to reversed_half: reversed_half = reversed_half │
│       × 10 + digit                                            │
│     - Remove last digit from x: x //= 10                     │
│                                                               │
│  4. Return:                                                   │
│     - Even digits: x == reversed_half                        │
│     - Odd digits: x == reversed_half // 10                   │
│                                                               │
│  Time: O(log n) - number of digits                           │
│  Space: O(1)                                                  │
└─────────────────────────────────────────────────────────────┘
```

**When to use**: Space-constrained environments, large numbers.

### Framework 2: Palindrome Generation

```
┌─────────────────────────────────────────────────────────────┐
│  GENERATE PALINDROME FROM HALF                                │
├─────────────────────────────────────────────────────────────┤
│  Input: Left half as integer, boolean odd_length              │
│  Output: Palindrome number                                    │
│                                                               │
│  1. Convert left half to string: s = str(left_half)          │
│                                                               │
│  2. If odd_length:                                           │
│     - Mirror excluding last digit: result = s + s[-2::-1]    │
│     Else:                                                     │
│     - Mirror entire string: result = s + s[::-1]             │
│                                                               │
│  3. Return int(result)                                        │
│                                                               │
│  Examples:                                                   │
│   - generate(12, odd=False) → "12" + "21" = 1221             │
│   - generate(12, odd=True) → "12" + "1" = 121                │
└─────────────────────────────────────────────────────────────┘
```

**When to use**: Finding next palindrome, generating palindromes in range.

### Framework 3: Next Palindrome Finder

```
┌─────────────────────────────────────────────────────────────┐
│  FIND NEXT PALINDROME                                         │
├─────────────────────────────────────────────────────────────┤
│  Input: Integer n                                            │
│  Output: Smallest palindrome greater than n                   │
│                                                               │
│  1. Convert n to string, find length                          │
│                                                               │
│  2. Extract left half: left = first (len+1)//2 digits        │
│                                                               │
│  3. Generate candidates:                                     │
│     - Same length with current left half                     │
│     - Same length with left half + 1                         │
│     - Next length (100...001)                                 │
│                                                               │
│  4. Return smallest candidate > n                             │
│                                                               │
│  Edge case: Handle carry that increases digit count            │
└─────────────────────────────────────────────────────────────┘
```

**When to use**: Finding closest palindrome problems.

## Tactics

Specific techniques and implementations.

### Tactic 1: Reverse Half (O(1) Space)

Most efficient numeric approach:

```python
def is_palindrome(x: int) -> bool:
    """
    Check if number is palindrome by reversing half.
    Time: O(log n), Space: O(1)
    """
    # Negative numbers and numbers ending in 0 (except 0) are not palindromes
    if x < 0 or (x % 10 == 0 and x != 0):
        return False
    
    reversed_half = 0
    while x > reversed_half:
        reversed_half = reversed_half * 10 + x % 10
        x //= 10
    
    # For even length: x == reversed_half
    # For odd length: x == reversed_half // 10 (middle digit doesn't matter)
    return x == reversed_half or x == reversed_half // 10
```

### Tactic 2: Full Number Reversal

Alternative O(1) space approach:

```python
def is_palindrome_full_reverse(x: int) -> bool:
    """
    Reverse entire number and compare.
    Time: O(log n), Space: O(1)
    """
    if x < 0:
        return False
    
    original = x
    reversed_num = 0
    while x > 0:
        # Check for overflow (optional, Python handles big ints)
        reversed_num = reversed_num * 10 + x % 10
        x //= 10
    
    return original == reversed_num
```

### Tactic 3: String Conversion

Simple but uses extra space:

```python
def is_palindrome_string(x: int) -> bool:
    """
    Convert to string and check.
    Time: O(log n), Space: O(log n)
    """
    if x < 0:
        return False
    s = str(x)
    return s == s[::-1]
```

### Tactic 4: Generate Palindrome from Half

Creating palindromes programmatically:

```python
def generate_palindrome(left_half: int, odd_length: bool = False) -> int:
    """
    Generate palindrome by mirroring left half.
    
    Args:
        left_half: The left part of the palindrome
        odd_length: If True, exclude middle digit when mirroring
    
    Returns:
        Generated palindrome number
    """
    s = str(left_half)
    if odd_length:
        return int(s + s[-2::-1])  # Exclude middle digit when mirroring
    else:
        return int(s + s[::-1])

# Examples:
# generate_palindrome(12, odd=False) → 1221
# generate_palindrome(12, odd=True) → 121
# generate_palindrome(123, odd=False) → 123321
# generate_palindrome(123, odd=True) → 12321
```

### Tactic 5: Find Next Palindrome

Finding the smallest palindrome greater than n:

```python
def next_palindrome(n: int) -> int:
    """
    Find smallest palindrome greater than n.
    Time: O(log n)
    """
    s = str(n)
    length = len(s)
    half = (length + 1) // 2
    left = int(s[:half])
    
    candidates = []
    
    # Try same length
    if length % 2 == 0:
        candidates.append(generate_palindrome(left, False))
        candidates.append(generate_palindrome(left + 1, False))
    else:
        candidates.append(generate_palindrome(left, True))
        candidates.append(generate_palindrome(left + 1, True))
    
    # Try next length (100...01)
    candidates.append(10 ** (length - 1) + 1)
    
    # Return smallest valid candidate
    for c in sorted(candidates):
        if c > n:
            return c
    
    return candidates[-1]
```

## Python Templates

### Template 1: Palindrome Number Check (Reverse Half)

```python
def is_palindrome(x: int) -> bool:
    """
    LeetCode 9: Palindrome Number.
    Check if integer reads same backward as forward.
    
    Args:
        x: Integer to check
    
    Returns:
        True if palindrome, False otherwise
        
    Time: O(log n) where n is the number
    Space: O(1)
    """
    # Negative numbers are not palindromes
    # Numbers ending in 0 (except 0 itself) are not palindromes
    if x < 0 or (x % 10 == 0 and x != 0):
        return False
    
    reversed_half = 0
    
    # Build reversed half until it exceeds or equals first half
    while x > reversed_half:
        reversed_half = reversed_half * 10 + x % 10
        x //= 10
    
    # Even length: both halves equal
    # Odd length: reversed_half//10 equals x (middle digit ignored)
    return x == reversed_half or x == reversed_half // 10
```

### Template 2: Palindrome Check (String)

```python
def is_palindrome_string(x: int) -> bool:
    """
    Check palindrome using string conversion.
    Simpler but uses O(log n) space.
    
    Args:
        x: Integer to check
    
    Returns:
        True if palindrome
        
    Time: O(log n)
    Space: O(log n)
    """
    if x < 0:
        return False
    
    s = str(x)
    left, right = 0, len(s) - 1
    
    while left < right:
        if s[left] != s[right]:
            return False
        left += 1
        right -= 1
    
    return True
```

### Template 3: Generate Palindrome

```python
def generate_palindrome(left_half: int, odd_length: bool = False) -> int:
    """
    Generate palindrome by mirroring left half.
    
    Args:
        left_half: Left portion of palindrome
        odd_length: True for odd-length palindrome
    
    Returns:
        Complete palindrome number
    """
    s = str(left_half)
    
    if odd_length:
        # Mirror excluding last character
        return int(s + s[-2::-1])
    else:
        # Mirror entire string
        return int(s + s[::-1])
```

### Template 4: Find Closest Palindrome

```python
def closest_palindrome(n: str) -> str:
    """
    LeetCode 564: Find the Closest Palindrome.
    Find closest palindrome to n (absolute difference).
    
    Args:
        n: Number as string (can be very large)
    
    Returns:
        Closest palindrome as string
    """
    length = len(n)
    half_len = (length + 1) // 2
    half = int(n[:half_len])
    
    candidates = set()
    
    # Same digit count
    candidates.add(generate_large_palindrome(half, length % 2 == 1))
    candidates.add(generate_large_palindrome(half + 1, length % 2 == 1))
    candidates.add(generate_large_palindrome(half - 1, length % 2 == 1))
    
    # Different digit count
    candidates.add(10 ** (length - 1) - 1)  # 999...9
    candidates.add(10 ** length + 1)       # 100...001
    
    # Remove n itself if it's a palindrome
    candidates.discard(int(n))
    
    # Find closest
    num = int(n)
    closest = None
    min_diff = float('inf')
    
    for c in candidates:
        diff = abs(c - num)
        if diff < min_diff or (diff == min_diff and c < closest):
            min_diff = diff
            closest = c
    
    return str(closest)

def generate_large_palindrome(left: int, odd: bool) -> int:
    """Generate palindrome for potentially large numbers."""
    s = str(left)
    if odd:
        return int(s + s[-2::-1])
    return int(s + s[::-1])
```

### Template 5: Count Palindromes in Range

```python
def count_palindromes(L: int, R: int) -> int:
    """
    Count palindrome numbers in range [L, R].
    Time: O(√R)
    """
    def count_up_to(n: int) -> int:
        if n < 0:
            return 0
        
        s = str(n)
        length = len(s)
        count = 0
        
        # Count palindromes with fewer digits
        for l in range(1, length):
            # For even length: 9 * 10^(l/2 - 1)
            # For odd length: 9 * 10^((l-1)/2)
            count += 9 * (10 ** ((l - 1) // 2))
        
        # Count palindromes with same length
        half_len = (length + 1) // 2
        left_half = int(s[:half_len])
        first_digit = int(s[0])
        
        # Palindromes with smaller first digit
        if first_digit > 1:
            count += (first_digit - 1) * (10 ** (half_len - 1))
        
        # Check palindromes with this left half
        for prefix in range(10 ** (half_len - 1), left_half + 1):
            if length % 2 == 0:
                palindrome = generate_palindrome(prefix, False)
            else:
                palindrome = generate_palindrome(prefix, True)
            if palindrome <= n:
                count += 1
        
        return count
    
    return count_up_to(R) - count_up_to(L - 1)
```

### Template 6: Valid Palindrome (String with Filtering)

```python
def is_valid_palindrome(s: str) -> bool:
    """
    LeetCode 125: Valid Palindrome.
    Check if string is palindrome considering only alphanumeric.
    
    Args:
        s: Input string
    
    Returns:
        True if palindrome after filtering
        
    Time: O(n)
    Space: O(1)
    """
    left, right = 0, len(s) - 1
    
    while left < right:
        # Skip non-alphanumeric from left
        while left < right and not s[left].isalnum():
            left += 1
        # Skip non-alphanumeric from right
        while left < right and not s[right].isalnum():
            right -= 1
        
        if s[left].lower() != s[right].lower():
            return False
        
        left += 1
        right -= 1
    
    return True
```

## When to Use

Use Palindrome detection/generation when you need to solve problems involving:

- **Number Validation**: Checking if numbers read the same backward
- **String Processing**: Palindromic substrings, longest palindrome
- **Mathematical Properties**: Palindromic primes, super palindromes
- **Closest Values**: Finding nearest palindrome to a number
- **Range Queries**: Counting palindromes in ranges

### Comparison: Numeric vs String Approaches

| Approach | Time | Space | Pros | Cons |
|----------|------|-------|------|------|
| **Reverse Half** | O(log n) | O(1) | Space efficient, no overflow | Slightly complex |
| **Full Reverse** | O(log n) | O(1) | Simple concept | Potential overflow |
| **String** | O(log n) | O(log n) | Very simple | Extra space |
| **Two Pointers** | O(n) | O(1) | Works for strings | Linear scan |

### When to Choose Each Approach

- **Choose Reverse Half** when:
  - Space is constrained
  - Working with very large numbers
  - Interview setting (demonstrates skill)

- **Choose String Conversion** when:
  - Code simplicity is priority
  - Not space constrained
  - Working in Python (handles big ints anyway)

- **Choose Two Pointers** when:
  - Processing strings with filtering
  - Need to handle non-alphanumeric characters
  - Longest palindrome substring problems

## Algorithm Explanation

### Core Concept

A palindrome reads the same forwards and backwards. For numbers, this means the first digit equals the last, the second equals the second-to-last, and so on. The reverse-half approach exploits this symmetry by comparing the first half with the reverse of the second half.

### How It Works

#### Step 1: Handle Edge Cases

```python
if x < 0 or (x % 10 == 0 and x != 0):
    return False
```
- Negative numbers: The minus sign makes them non-palindromic
- Trailing zeros: Can't have leading zeros, so 10, 100, etc. aren't palindromes

#### Step 2: Reverse Half

For x = 1221:
```
Iteration 1: reversed_half = 1, x = 122
Iteration 2: reversed_half = 12, x = 12
Stop: x <= reversed_half

Compare: 12 == 12 ✓ (even length, direct match)
```

For x = 12321:
```
Iteration 1: reversed_half = 1, x = 1232
Iteration 2: reversed_half = 12, x = 123
Iteration 3: reversed_half = 123, x = 12
Stop: x < reversed_half

Compare: 12 == 123 // 10 = 12 ✓ (odd length, ignore middle)
```

### Visual Representation

**Even Length Palindrome (1221)**:
```
Original:  1  2 | 2  1
           ↓  ↓   ↓  ↓
Position:  1  2   3  4

Reverse second half: 12 | 12 ✓ Match!
```

**Odd Length Palindrome (12321)**:
```
Original:  1  2 | 3 | 2  1
           ↓  ↓     ↓  ↓
Position:  1  2  3  4  5

Reverse second half: 12 | 3 | 12
After removing middle: 12 | 12 ✓ Match!
```

### Why It Works

1. **Symmetry**: Palindromes are symmetric around their center
2. **Half Comparison**: Comparing half is sufficient due to symmetry
3. **Digit Extraction**: Modulo and division extract and remove digits efficiently
4. **Odd/Even Handling**: Integer division handles both cases elegantly

### Limitations

- **Overflow**: In languages with fixed-size integers, full reversal might overflow
- **Negative Numbers**: Must be explicitly handled
- **Single Digit**: Edge case handled correctly (always palindrome)

## Follow-up Questions

### Q1: Could you solve it without converting the integer to a string?

**Answer:** Yes, the reverse-half approach does exactly this. It extracts digits mathematically using modulo and division, building the reversed half without any string conversion. This gives O(1) space complexity compared to O(log n) for string approaches.

---

### Q2: How do you handle overflow when reversing numbers?

**Answer:** The reverse-half approach naturally avoids overflow because we only reverse half the digits. Even for a 32-bit integer, half the digits (at most 5) fit easily. For full reversal, check if reversed > (2^31 - 1) / 10 before appending the next digit, or use a larger data type.

---

### Q3: Why are numbers ending in 0 (except 0 itself) not palindromes?

**Answer:** A number ending in 0 would need to start with 0 to be a palindrome, but standard number representation doesn't have leading zeros. So 10 reversed is 01 = 1, not 10. Only 0 itself is a valid single-digit palindrome ending in 0.

---

### Q4: How do you count palindromes in a range efficiently?

**Answer:** Use the property that palindromes are determined by their first half. For range [L, R], count palindromes ≤ R minus count ≤ L-1. For each digit length, there are 9 × 10^((len-1)/2) palindromes (9 choices for first digit, then free choices for remaining half).

---

### Q5: What's the time complexity of palindrome operations?

**Answer:**
- Checking: O(log n) - proportional to number of digits
- Generating next: O(log n) - manipulate half the digits
- Counting in range: O(√R) - enumerate first halves up to √R
- Finding k-th: O(k) or O(log k) with binary search

## Summary

Palindrome number problems combine mathematical insight with careful implementation. The key takeaways are:

1. **Reverse Half**: O(1) space solution using mathematical digit manipulation
2. **Edge Cases**: Handle negatives, trailing zeros, and single digits
3. **Generation**: Mirror left half to create palindromes
4. **Counting**: Use digit DP or enumeration via first half
5. **String Alternative**: Simpler but uses extra space

**When to Use:**
- Number validation problems
- Palindrome generation requirements
- Closest palindrome searches
- Mathematical palindrome properties

**Key Formula:**
```
Palindrome check: O(log n) time, O(1) space
Generation from half: mirror s to create s + s[::-1]
Counting: ~2 × √N palindromes ≤ N
```

This pattern appears frequently in interviews and competitive programming, testing both mathematical reasoning and careful edge case handling.
